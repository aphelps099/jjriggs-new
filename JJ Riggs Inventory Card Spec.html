1 Card Anatomy — Annotated

Each inventory card is built from 6 distinct zones. The annotated example below shows a card in the "Limited" status with a "Just In" tag — the most visually complex state. All other states simplify from this.

  • 1 Tag Badge (corner ribbon) Pill badge, top-left of image. Set by the "Tag" column in the Sheet. Always on top of image. Use sparingly — max 1 tag per card.
  • 2 Status Badge Colored dot + label. Top-right overlaid on image for "Just In" cards; in card body otherwise. Color-coded to status (green/amber/orange/blue/purple/grey).
  • 3 Category + SKU Faint uppercase label above the name. SKU in monospace below. Both are metadata — never bolder than the product name.
  • 4 Product Name Barlow Condensed Bold, the dominant text element. Size: --text-lg. Max 2 lines — truncate beyond that.
  • 5 Qty / Availability Note Only shown when qty < 5 or when status is Backorder/Custom. "3 left on lot" or "Est. 2–3 weeks". Red text when qty ≤ 2.
  • 6 CTA Footer Separated by a hairline divider. Primary action changes based on status — see Section 4.

3 Tag System — Live Inventory Feel

Tags are set in the "Tag" column of the Sheet. They appear as pill badges in the top-left corner of the image. One tag per card maximum. Tags create urgency and personality without changing the card's fundamental status.

Developer note: Tags and Statuses are separate fields. A card can have Status = "In Stock" and Tag = "Just In" simultaneously. The tag always renders on top of the image; the status badge is in the card body.

All tag variants

Just In

New arrival. Use when freshly added to lot.

🔥 2 Left

Urgency. Use when qty is 1–3. Text = "X Left" where X = qty.

⭐ Popular

Social proof. Manually applied to top-sellers.

❄ Seasonal

Contextual. Apply to snow equip in fall/winter.

🏷 Clearance

Price reduction. Always show alongside price if visible.

4 CTA Logic — Button Decision Tree

The primary CTA button changes based on Status. The secondary "Details" button is always present on In Stock and Limited items. The developer should implement this as a simple if/else on the Status field.

Implementation note: All CTA clicks should open a contact form, email link (mailto:), or phone number — not a shopping cart. This is a quote-based business.
Status Primary Button Style Secondary Sub-note
In Stock "Get a Quote" Solid Green "Details" ghost btn —
Limited "Get a Quote" Solid Green "Details" ghost btn —
Low Stock "Get a Quote" Solid Green "Details" ghost btn Consider urgency copy
Backorder "Reserve It" Outline Green Optional: "Details" Show lead time note below btn
Custom Order "Request a Quote" Outline Green None Add small text: "Available to order"
Discontinued "No Longer Available" Disabled grey None Card at 55% opacity + grayscale
